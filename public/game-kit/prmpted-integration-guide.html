prmpted Games / Integration Guide

Everything you need to ship a browser game to prmpted.
This guide is paired with prmpted.md in the same kit. Hand the markdown file to your AI coding agent — it contains the same information formatted for an LLM to act on.

1. Will my game work?

prmpted Games embeds your game in a sandboxed iframe. To work, it must be 100% client-side:

  • Static HTML / JS / CSS / assets only
  • No backend, no Node server, no /api/* calls to your own domain
  • External APIs (Firebase, public REST) are fine if they support CORS

3. Framework setup

Vite

// vite.config.ts
export default defineConfig({
  base: './',   // <- this line
  // ...
});

Then npm run build. Your build output (dist/) is what you host or zip.

React Router

Use HashRouter, not BrowserRouter — the sandbox does not serve fallback routes.

import { HashRouter, Routes, Route } from 'react-router-dom';

<HashRouter>
  <Routes>
    <Route path="/" element={<Lobby />} />
    <Route path="/play" element={<Game />} />
  </Routes>
</HashRouter>

Create React App

// package.json
{ "homepage": "." }

Next.js

Not directly supported — needs a server. Use next export only if your project has no server-side features.

4. The SDK

Paste this at the top of <head> in your index.html. No keys. No setup. Does nothing unless embedded in prmpted.

<script>
(function(){
  if (window.prmpted) return;
  function send(m){ try { window.parent.postMessage(m, '*'); } catch(_){} }
  function heartbeat(){
    if (document.visibilityState !== 'visible') return;
    send({ type: 'prmpted:heartbeat', t: Date.now() });
  }
  setInterval(heartbeat, 30000);
  setTimeout(heartbeat, 2000);
  window.prmpted = {
    unlock: function(id){
      if (!id || typeof id !== 'string') return;
      send({ type: 'prmpted:achievement', id: id });
    }
  };
})();
</script>

5. Achievements (+15 BP each)

Call window.prmpted.unlock('your_id') whenever a player earns an achievement. First unlock per player awards +15 Builder Points. Repeats are deduped automatically.

// inside your game's achievement function
function unlockAch(id) {
  // ... your existing logic ...
  try {
    if (window.prmpted && window.prmpted.unlock) window.prmpted.unlock(id);
  } catch(_){}
}

Choosing IDs

Use stable, snake_case strings. They are the identity used for dedup; renaming after launch orphans existing unlocks.

  • first_blood
  • level_10_complete
  • no_damage_run
  • secret_room_found

6. Save state

If your game uses localStorage / sessionStorage, it just works. prmpted bridges storage calls to the parent and persists them per-user (Supabase if logged in, browser localStorage if anon). No code required.

8. Pre-flight checklist

  • ☐ Built with base: './' (or framework equivalent)
  • ☐ Open built index.html by double-clicking — it works without a server
  • ☐ DevTools shows no 404s for assets
  • ☐ SDK <script> is present in the built index.html, not just source
  • ☐ Every achievement-grant site calls window.prmpted.unlock(...)
  • ☐ Under 200 MB if zipping
  • ☐ Thumbnail ready (1280×720 PNG/JPG)

10. Submit

Open the Games tab → Upload Game. Choose URL or Zip mode. Fill in title, description, the achievement IDs you wired up, and your AI-tool disclosure. A human reviewer approves before it appears in the public grid.
